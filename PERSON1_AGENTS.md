You own `/backend/agents/`. You build the LangGraph state machines, tool definitions, and system prompts that power the 3 AI agents.

- `backend/agents/state.py` — State schemas for both graphs

### Prompt 3 — Graph 1 (Patient Chat)

Open `backend/agents/graph_patient.py` and tell Claude Code:

```
Build a LangGraph StateGraph for the patient chat pipeline.

Flow: START → whisper_stt (conditional, only if voice) → sealion_understand 
→ agent1_reason → execute_tools ←→ loop back if more tools 
→ agent3_monitor → sealion_localise → END

Node functions:
1. whisper_stt: if input_type=="voice", call services.whisper_service.transcribe_audio()
2. sealion_understand: call services.sealion_service.understand_input() to detect language and extract intent
3. agent1_reason: call services.claude_service.chat_with_tools() with AGENT1_SYSTEM_PROMPT and TOOL_DEFINITIONS
4. execute_tools: loop through tool_calls and call agents.tools.execute_tool() for each
5. agent3_monitor: call services.claude_service.check_anomalies() with AGENT3_SYSTEM_PROMPT, write alerts to DB
6. sealion_localise: call services.sealion_service.localise_response() to translate English response

Routing:
- Entry: if input_type=="voice" → whisper_stt, else → sealion_understand
- After agent1_reason: if tool_calls not empty → execute_tools, else → agent3_monitor
- After execute_tools: loop back to agent1_reason (Claude may need multiple tool rounds)

Use set_conditional_entry_point and add_conditional_edges.
Import state from agents.state, compile the graph, export as patient_chat_graph.
```

### Prompt 5 — System Prompts

Open `backend/agents/prompts.py` and tell Claude Code:

```
Create system prompts for 3 AI agents in a diabetes management system.

AGENT1_SYSTEM_PROMPT (Patient Companion):
- Warm, empathetic, like a caring Singaporean friend
- Knows local hawker food (char kway teow, nasi lemak, chicken rice, kopi)
- Suggests local healthier alternatives (brown rice bee hoon, kopi-O kosong, fish soup)
- Never judgmental about food choices — acknowledge before advising
- Uses tools to log meals, meds, glucose, check goals/schedule/referrals
- Proactively references doctor's lifestyle goals when relevant
- Responds in English (SEA-LION handles translation after)
- Never prescribes or diagnoses — only suggests and reminds
- Keep responses to 2-4 sentences

AGENT2_SYSTEM_PROMPT (Clinical Analyst):
- Clinical, precise, evidence-based tone for a GP
- Outputs structured JSON: summary, key_findings, recommendations, risk_level
- Analyses glucose control, medication adherence, dietary patterns, goal compliance
- References specific data points with dates
- Knows Singaporean food nutritional impact

AGENT3_SYSTEM_PROMPT (Proactive Monitor):
- Conservative — only flag genuinely concerning patterns
- Checks: glucose >10 or <4, missed meds, high-carb meals, goal noncompliance
- Outputs JSON array of alerts with type, severity (info/warning/critical), title, description
- Severity guide: info=minor, warning=pattern forming, critical=immediate attention
```
